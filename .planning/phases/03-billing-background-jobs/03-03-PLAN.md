1. Create `packages/modules/billing/src/commands/create-checkout-session.ts` per D-04:
   - Input schema: `{ priceId: Type.String(), successUrl: Type.String(), cancelUrl: Type.String() }`
   - Look up billing_customers by ctx.tenantId to get stripeCustomerId
   - If no billing customer found, return err("BILLING_NOT_CONFIGURED")
   - Call `stripe.checkout.sessions.create({ customer: stripeCustomerId, mode: "subscription", line_items: [{ price: input.priceId, quantity: 1 }], success_url: input.successUrl, cancel_url: input.cancelUrl })` with idempotency key via `crypto.randomUUID()` (D-09)
   - Return ok({ sessionId: session.id, url: session.url })

2. Create `packages/modules/billing/src/commands/cancel-subscription.ts` per D-05:
   - Input schema: `{}` (empty -- uses tenant context)
   - Look up billing_customers by ctx.tenantId to get stripeSubscriptionId
   - If no active subscription, return err("NO_ACTIVE_SUBSCRIPTION")
   - Call `stripe.subscriptions.update(subscriptionId, { cancel_at_period_end: true })` with idempotency key (D-09)
   - Return ok({ cancelledAt: "period_end", currentPeriodEnd: billingCustomer.currentPeriodEnd })

3. Create `packages/modules/billing/src/commands/change-subscription.ts` per D-05:
   - Input schema: `{ newPriceId: Type.String() }`
   - Look up billing_customers by ctx.tenantId to get stripeSubscriptionId
   - If no active subscription, return err("NO_ACTIVE_SUBSCRIPTION")
   - Retrieve subscription from Stripe to get the current subscription item ID: `const sub = await stripe.subscriptions.retrieve(subscriptionId)`
   - Call `stripe.subscriptions.update(subscriptionId, { items: [{ id: sub.items.data[0].id, price: input.newPriceId }] })` with idempotency key (D-09)
   - Return ok({ subscriptionId, newPriceId: input.newPriceId })

4. Create `packages/modules/billing/src/commands/create-one-time-payment.ts` per D-06:
   - Input schema: `{ priceId: Type.String(), quantity: Type.Optional(Type.Number({ minimum: 1, default: 1 })), successUrl: Type.String(), cancelUrl: Type.String() }`
   - Look up billing_customers by ctx.tenantId to get stripeCustomerId
   - Call `stripe.checkout.sessions.create({ customer: stripeCustomerId, mode: "payment", line_items: [{ price: input.priceId, quantity: input.quantity ?? 1 }], success_url: input.successUrl, cancel_url: input.cancelUrl })` with idempotency key (D-09)
   - Return ok({ sessionId: session.id, url: session.url })

5. Create `packages/modules/billing/src/commands/create-portal-session.ts` per D-08:
   - Input schema: `{ returnUrl: Type.String() }`
   - Look up billing_customers by ctx.tenantId to get stripeCustomerId
   - Call `stripe.billingPortal.sessions.create({ customer: stripeCustomerId, return_url: input.returnUrl })`
   - Return ok({ url: session.url })

All commands:

- Access `ctx.db` for tenant-scoped queries (but billing_customers may need unscoped access via createDb since stripeCustomerId is not tenant-scoped in query context -- check if ctx.db works for billing_customers which has tenantId column)
- Add `crypto.randomUUID()` as idempotency key header: pass `{ idempotencyKey: crypto.randomUUID() }` as the options parameter to Stripe calls

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> billing API routes | Tenant-scoped requests, must be authenticated |
| API -> Stripe API | Outbound calls with secret key, needs idempotency |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-10 | Elevation of Privilege | Billing commands | mitigate | All billing commands use HandlerContext.tenantId for scoping; commands only operate on the requesting tenant's billing_customers record (WHERE tenantId = ctx.tenantId) |
| T-03-11 | Tampering | Stripe API calls | mitigate | Idempotency keys (crypto.randomUUID()) on all mutation API calls per D-09; prevents duplicate charges from retry logic |
| T-03-12 | Information Disclosure | Billing history query | mitigate | get-billing-history uses ctx.tenantId to look up stripeCustomerId; returns only the tenant's own invoices |
| T-03-13 | Denial of Service | Stripe API rate limits | accept | Stripe has built-in rate limits; commands are user-initiated (not automated); low risk |
</threat_model>
