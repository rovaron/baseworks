<objective>
Refactor `apps/web/components/invite-dialog.tsx` to use the shared `Form`/`FormField`/`FormItem`/`FormLabel`/`FormControl`/`FormMessage` primitives from `@baseworks/ui`. This closes the GAP-3 regression flagged by the v1.1 audit where the dialog bypassed the shared primitives and rendered errors as raw `<p className="text-sm text-destructive">`, bypassing the `role="alert"` and `aria-describedby` wiring that satisfies A11Y-04 and A11Y-05.

Purpose: `FormMessage` already renders `role="alert"` (form.tsx:158) and `FormControl` already wires `aria-describedby` + `aria-invalid` (form.tsx:114-119). By routing the InviteDialog fields through these primitives, the a11y wiring comes for free and the existing `packages/ui/src/components/__tests__/form.a11y.test.tsx` vitest-axe suite transitively protects the dialog. The existing validation branching logic (`errors.email.type === "too_small"` vs invalid email) is moved from the JSX into the zod schema so `FormMessage` renders `errors.email.message` without any conditional logic.

Output: One modified file — `apps/web/components/invite-dialog.tsx` — with the form refactored to use Form primitives and the schema carrying i18n messages directly. All visible behavior, layout, and translation keys are preserved.
</objective>
