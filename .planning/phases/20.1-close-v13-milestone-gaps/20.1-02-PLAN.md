<objective>
Diagnose and fix the `TypeError: undefined is not an object (evaluating 'table[Table.Symbol.Columns]')` thrown by `GET /api/billing/subscription` for tenants with no `billing_customers` row, and lock the fix with an API-level integration regression test. Closes Phase 20.1 SC#2.

Purpose: A freshly-registered tenant hitting the billing subscription endpoint currently triggers a 500. The leading hypothesis (CONTEXT.md D-05) is that the `billingCustomers` symbol re-exported via `packages/modules/billing/src/schema.ts` is `undefined` at handler invocation time due to workspace import resolution timing — Drizzle's `select().from(billingCustomers)` then crashes when reading `Table.Symbol.Columns` off `undefined`. D-05 confirms via a 1-line probe; D-07 fixes by importing from `@baseworks/db` directly.

Output:
- One probe diagnostic (verified, then deleted)
- Direct import in `get-subscription-status.ts` from `@baseworks/db`
- `billingCustomers` re-export removed from `packages/modules/billing/src/schema.ts`
- New integration test `apps/api/__tests__/billing-subscription.test.ts` asserting the 200 + body shape for the no-billing-row case
</objective>
