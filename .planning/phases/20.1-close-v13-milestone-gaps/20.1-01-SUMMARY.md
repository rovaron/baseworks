## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] drizzle.config.ts paths broke `db:migrate` when invoked from repo root**
- **Found during:** Task 2 GREEN execution (`bun run db:generate` and `bun run db:migrate` from repo root)
- **Issue:** `drizzle.config.ts` declared `schema: "./src/schema/index.ts"` and `out: "./migrations"` resolved relative to cwd. When the package.json scripts (`bun drizzle-kit generate --config packages/db/drizzle.config.ts`) ran from the repo root, drizzle-kit looked for `./src/schema/index.ts` at the repo root and failed.
- **Fix:** Anchored both paths to the package root via workspace-relative absolute resolution. `bun run db:generate` and `bun run db:migrate` now work from any cwd, including the repo root (which is what the regression test calls into via Bun.spawn).
- **Files modified:** `packages/db/drizzle.config.ts`
- **Verification:** Operator-verified Task 3 schema push (`bun run db:migrate` from repo root against fresh Docker Postgres exited 0 and produced the expected schema). Required for SC#1.
- **Committed in:** `b3759b6` (GREEN gate fix, folded with #2 below)

**2. [Rule 1 - Bug] Fresh-clone test left dangling Postgres connection pools**
- **Found during:** Task 2 GREEN execution (early test runs fluky)
- **Issue:** The regression test created `createDb(ADMIN_DB_URL)` in `beforeAll` and `afterAll` but did not close the underlying `postgres` (postgres.js) pool. The dangling connection blocked `DROP DATABASE baseworks_migrate_test` (cannot drop a DB while connections are open), causing intermittent afterAll failures.
- **Fix:** Test now opens an explicit pool, runs its work, and closes the pool before issuing the DROP. Mirrors the connection.test.ts pattern.
- **Files modified:** `packages/db/src/__tests__/fresh-clone-migrate.test.ts`
- **Verification:** Test runs cleanly when Postgres available; correctly skips when not. Required for D-03 / SC#1.
- **Committed in:** `b3759b6` (GREEN gate fix)
