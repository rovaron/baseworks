## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal padding |
| sm | 8px | Compact element spacing, card internal gaps |
| md | 16px | Default element spacing, card padding, sidebar item padding |
| lg | 24px | Section padding, content area gutters |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page-level vertical spacing |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets: minimum 44px (11 * 4px) height for all interactive elements on mobile, per existing `min-h-11` pattern in sidebar nav items
- Sidebar widths: 256px (16rem) expanded, 48px (3rem) icon rail — already set as `--sidebar-width: 16rem` and `--sidebar-width-icon: 3rem` in sidebar.tsx

Source: Pre-existing codebase conventions (sidebar.tsx line 30-32)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 (leading-normal) |
| Label | 12px (text-xs) | 500 (medium) | 1.5 |

Phase-specific notes:
- Mobile data table cards use Label (12px, weight 500) for column headers and Body (14px, weight 400) for values
- Sort dropdown and filter chips use Label size (12px)
- No Display size needed — this phase adds no new page-level headers

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | hsl(0 0% 100%) / `--background` | Page background, main content area |
| Secondary (30%) | hsl(0 0% 98%) / `--sidebar-background` | Sidebar surface, mobile card backgrounds, card containers |
| Accent (10%) | hsl(0 0% 9%) / `--primary` | Sidebar toggle button active state, sort-by dropdown active indicator, expand/collapse chevron |
| Destructive | hsl(0 84.2% 60.2%) / `--destructive` | Not used in this phase — no destructive actions introduced |

Accent reserved for:
- Active sidebar nav item highlight (via `--sidebar-primary`)
- Hamburger menu icon (mobile trigger)
- Sort-by dropdown selected state indicator
- Card expand/collapse chevron icon

Additional color notes:
- Card borders: `--border` (hsl(0 0% 89.8%))
- Muted column labels in cards: `--muted-foreground` (hsl(0 0% 45.1%))
- Sidebar collapsed icon rail uses `--sidebar-accent` for hover state

Source: `packages/ui/src/styles/globals.css` existing CSS custom properties

## Interaction Contracts

### Sidebar — Desktop (>= 1024px)

| Interaction | Behavior |
|-------------|----------|
| Page load | Sidebar expanded (256px) by default, or restored from localStorage |
| Click toggle button | Collapse to icon rail (48px) or expand to full; persist to localStorage |
| Keyboard shortcut | Ctrl+B (existing) toggles sidebar |
| Transition | Width animates over 200ms ease-in-out (existing CSS transition) |

### Sidebar — Tablet (768px - 1023px)

| Interaction | Behavior |
|-------------|----------|
| Page load | Icon-only rail (48px) by default |
| Hover on rail (mouse) | Expand to full width after 200ms delay; collapse when mouse leaves |
| Tap on rail item (touch) | Navigate directly to that route; tooltip shows label |
| Transition | Width animates over 200ms ease-in-out |

### Sidebar — Mobile (< 768px)

| Interaction | Behavior |
|-------------|----------|
| Page load | Sidebar hidden; hamburger icon visible in top bar |
| Tap hamburger | Sheet drawer slides in from left (18rem wide) with overlay backdrop |
| Tap nav item | Navigate to route, Sheet auto-dismisses |
| Tap overlay | Sheet dismisses |
| Swipe left | Sheet dismisses (Radix Sheet built-in) |

### Data Table Cards — Mobile (< 768px)

| Interaction | Behavior |
|-------------|----------|
| View | Each row renders as a Card showing 2-3 priority columns |
| Tap card | Expands inline to show all remaining columns below the priority fields |
| Tap expanded card | Collapses back to summary |
| Sort | Select dropdown above card list; options derived from sortable columns |
| Filter | Filter chips above card list (same filters as desktop table) |

## Mobile Card Layout Specification

```
+--------------------------------------+
| [Priority 1 value]     [Status badge]|
| Priority 2 label: Priority 2 value   |
|                              [chevron]|
+--------------------------------------+

Expanded:
+--------------------------------------+
| [Priority 1 value]     [Status badge]|
| Priority 2 label: Priority 2 value   |
|---------------------------------------|
| Label 3: Value 3                      |
| Label 4: Value 4                      |
| Label 5: Value 5                      |
|                              [chevron]|
+--------------------------------------+
```

- Card padding: 16px (md)
- Gap between cards: 12px (3 * 4px)
- Priority 1 field: 14px semibold (card title role)
- Priority 2+ fields: 12px label + 14px value
- Expanded detail fields: 12px label (muted-foreground) + 14px value
- Separator between summary and detail: 1px `--border` color
- Card background: `--card` (white)
- Card border: 1px solid `--border`
- Card border-radius: `--radius-lg` (0.5rem)
- Chevron icon: 16px, `--muted-foreground`, rotates 180deg when expanded

## Page-Level Responsive Rules

### Spacing Adjustments by Breakpoint

| Element | Mobile (< 768px) | Tablet (768-1023px) | Desktop (>= 1024px) |
|---------|-------------------|---------------------|----------------------|
| Page content padding | 16px (px-4) | 24px (px-6) | 32px (px-8) |
| Section vertical gap | 24px (gap-6) | 24px (gap-6) | 32px (gap-8) |
| Grid columns (detail pages) | 1 column | 1 column | 2 columns (lg:grid-cols-2) |
| Form fields | Full width stacked | Full width stacked | Side-by-side where logical |

### Stacking Rules

- Grid layouts with `lg:grid-cols-2` or `lg:grid-cols-3` collapse to single column below `lg` (1024px)
- Stat cards / KPI cards: stack vertically on mobile, 2-col on tablet, 3-4 col on desktop
- Billing plans grid: stack vertically on mobile, horizontal on desktop (existing `flex-col lg:flex-row`)

| Tablet sidebar hover expand | width | 200ms | ease-in-out | Mouse enter (200ms delay) |
