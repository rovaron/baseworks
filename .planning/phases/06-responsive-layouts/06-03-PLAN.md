**Admin Dashboard Pages (RESP-06):**

**3. Billing overview (`apps/admin/src/routes/billing/overview.tsx`)**
- Stat cards grid already uses `grid grid-cols-1 lg:grid-cols-3 gap-6`. Good.
- The "Recent Subscriptions" section uses a raw `<Table>` (not DataTable). This has 4 columns (Tenant, Plan, Status, Date) which can overflow at 375px.
- Solution: Convert to a stacked layout on mobile. Use Tailwind responsive utilities:
  - Keep the Table for desktop: wrap the entire `<Table>` in `<div className="hidden md:block">`
  - Add a mobile view: `<div className="block md:hidden space-y-3">` that maps `recentSubscriptions` into small cards:
    ```tsx
    <div className="block md:hidden space-y-3">
      {recentSubscriptions.slice(0, 10).map((sub: any, i: number) => (
        <div key={sub.id ?? i} className="flex items-center justify-between rounded-lg border p-3">
          <div className="space-y-1">
            <p className="text-sm font-medium">{sub.tenantName ?? sub.tenantId}</p>
            <p className="text-xs text-muted-foreground">{sub.plan} &middot; {sub.date ? new Date(sub.date).toLocaleDateString() : "N/A"}</p>
          </div>
          <Badge variant={sub.status === "active" ? "default" : "secondary"}>
            {sub.status}
          </Badge>
        </div>
      ))}
    </div>
    ```
  This renders both views in DOM but CSS hides the unused one. Since max 10 items, the extra DOM is negligible.
