**@baseworks/shared** (per D-02, D-05, D-07, D-08):

`packages/shared/src/types/module.ts`:
```typescript
import type { Elysia } from 'elysia'  // Type-only import -- no runtime Elysia dep in shared
import type { CommandHandler, QueryHandler } from './cqrs'

export interface JobDefinition {
  queue: string
  handler: (data: unknown) => Promise<void>
}

export interface ModuleDefinition {
  name: string
  routes?: (app: Elysia<any>) => Elysia<any>
  commands?: Record<string, CommandHandler<any, any>>
  queries?: Record<string, QueryHandler<any, any>>
  jobs?: Record<string, JobDefinition>
  events?: string[]
}
```

NOTE: Elysia is a type-only import in `module.ts`. The `@baseworks/shared` package does NOT have Elysia as a runtime dependency -- the import is `import type` only. This preserves the anti-pattern rule from Research: "shared packages must NOT depend on Elysia."

`packages/shared/src/types/cqrs.ts` (per D-05, D-07, D-08):
- Export `Result<T>` type: `{ success: true; data: T } | { success: false; error: string }`
- Export `HandlerContext` interface: `{ tenantId: string; userId?: string; db: any; emit: (event: string, data: unknown) => void; enqueue?: (job: string, data: unknown) => Promise<void> }`
  - Note: `db` is typed as `any` here. The actual `ScopedDb` type will be defined in `@baseworks/db` (Plan 03). Using `any` avoids a circular dependency between shared and db packages. This is acceptable because handlers receive the db through dependency injection, not by importing it.
- Export `CommandHandler<TInput, TOutput>` type: `(input: TInput, ctx: HandlerContext) => Promise<Result<TOutput>>`
- Export `QueryHandler<TInput, TOutput>` type: `(input: TInput, ctx: HandlerContext) => Promise<Result<TOutput>>`
- Export `defineCommand(schema, handler)` factory function using TypeBox `TypeCompiler.Compile(schema)` for validation. Returns a wrapped handler that validates input before calling the real handler. On validation failure, returns `{ success: false, error: 'VALIDATION_ERROR: ...' }`.
- Export `defineQuery(schema, handler)` factory function -- same pattern as defineCommand.
- Import `TypeCompiler` from `@sinclair/typebox/compiler` and `Type`, `Static`, `TSchema` from `@sinclair/typebox`.

`packages/shared/src/types/context.ts`:
- Export `TenantContext` type: `{ tenantId: string }`
- Export `AppContext` extending TenantContext: adds `userId?: string`

`packages/shared/src/types/events.ts` (per D-06):
- Export `DomainEvents` interface -- initially empty, modules will extend via declaration merging. Include a comment explaining how to extend:
```typescript
// Extend in module files:
// declare module '@baseworks/shared' {
//   interface DomainEvents {
//     'user.created': { id: string; tenantId: string }
//   }
// }
export interface DomainEvents {
  [key: string]: Record<string, unknown>
}
```

`packages/shared/src/result.ts`:
- Export `ok<T>(data: T): Result<T>` helper -- returns `{ success: true, data }`
- Export `err(error: string): Result<never>` helper -- returns `{ success: false, error }`
- These are convenience constructors so handlers don't have to write the object literal every time.

`packages/shared/src/index.ts`: Barrel export everything from types/* and result.ts.
