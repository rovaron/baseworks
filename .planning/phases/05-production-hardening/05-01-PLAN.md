Per D-01, D-02, D-03: Create three multi-stage Dockerfiles using `oven/bun` as base image.

**Dockerfile.api** (multi-stage):
- Stage 1 "install": FROM oven/bun:1 AS install, WORKDIR /app, copy package.json, bun.lock, all workspace package.json files (packages/*/package.json, packages/modules/*/package.json, apps/*/package.json), run `bun install --frozen-lockfile`
- Stage 2 "app": FROM oven/bun:1, WORKDIR /app, copy node_modules from install stage, copy entire monorepo source (per D-02), set ENV NODE_ENV=production, EXPOSE 3000, CMD ["bun", "run", "apps/api/src/index.ts"]
- Add HEALTHCHECK: `--interval=30s --timeout=5s --retries=3 CMD ["bun", "-e", "fetch('http://localhost:3000/health').then(r => r.ok ? process.exit(0) : process.exit(1)).catch(() => process.exit(1))"]`

**Dockerfile.worker** (multi-stage):
- Same install stage pattern as Dockerfile.api
- Stage 2: same base, but CMD ["bun", "run", "apps/api/src/worker.ts"]
- Add ENV INSTANCE_ROLE=worker
- No HEALTHCHECK yet (worker health server created in Plan 02; HEALTHCHECK will be added to docker-compose.yml after Plan 02)

**Dockerfile.admin** (multi-stage):
- Stage 1 "build": FROM oven/bun:1 AS build, WORKDIR /app, copy all package files, run bun install, copy source, run `bun run build:admin` (produces apps/admin/dist/)
- Stage 2 "serve": FROM nginx:alpine, copy apps/admin/dist/ from build stage to /usr/share/nginx/html, copy a custom nginx.conf that handles SPA routing (try_files $uri $uri/ /index.html), EXPOSE 80

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Docker network -> host | Container ports exposed to host network |
| .env -> Docker | Secrets passed via environment variables |
| nginx -> client | Admin SPA served as static files |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | I (Information Disclosure) | docker-compose.yml | mitigate | Default BETTER_AUTH_SECRET in compose is for local dev only; production must override via .env file. Document this clearly. |
| T-05-02 | I (Information Disclosure) | .dockerignore | mitigate | Exclude .env, .env.local, .planning/, .git/ from Docker context via .dockerignore |
| T-05-03 | T (Tampering) | Dockerfile.admin nginx | mitigate | nginx serves only static files from /usr/share/nginx/html; no server-side execution |
| T-05-04 | S (Spoofing) | Docker network | accept | Local dev only; production deployment uses separate infrastructure with proper network isolation |
</threat_model>
