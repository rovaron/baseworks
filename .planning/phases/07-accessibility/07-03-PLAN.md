1. **Install vitest-axe** (per D-10):
   ```bash
   cd packages/ui && bun add -d vitest-axe
   ```

2. **Update test setup** in `packages/ui/src/test-setup.ts`:
   Add `import "vitest-axe/extend-expect";` after the existing jest-dom import.
   Final file:
   ```typescript
   import "@testing-library/jest-dom/vitest";
   import "vitest-axe/extend-expect";
   ```

3. **Create shared axe config helper** -- in each test file, use configureAxe with impactLevels filtering (per D-11):
   ```typescript
   import { configureAxe } from "vitest-axe";
   const axe = configureAxe({
     impactLevels: ["critical", "serious"],
   });
   ```
   This ensures tests only fail on critical and serious violations. Minor and moderate are warnings only.

   **NOTE on configureAxe API:** If `configureAxe` does not accept `impactLevels` option (Assumption A2 in RESEARCH.md), use the standard `axe()` function and filter results manually:
   ```typescript
   import { axe } from "vitest-axe";
   const results = await axe(container);
   const serious = results.violations.filter(v => v.impact === "critical" || v.impact === "serious");
   expect(serious).toHaveLength(0);
   ```

4. **Create test files** (per D-10 -- packages/ui components only):

   **`button.a11y.test.tsx`:**
   - Test Button with text content has no violations
   - Test Button with `aria-label` (icon-only) has no violations
   - Test disabled Button has no violations

   **`dialog.a11y.test.tsx`:**
   - Test Dialog (open state) with DialogTitle and DialogDescription has no violations
   - Must render Dialog with `open={true}` to test the portal content
   - Include DialogTitle (required for accessible dialog)

   **`form.a11y.test.tsx`:**
   - Test FormField with FormLabel + FormControl + FormMessage (with error) has no violations
   - Requires wrapping in FormProvider from react-hook-form (use `useForm` to create context)
   - Verify FormMessage renders with `role="alert"` when error is present

   **`skip-link.a11y.test.tsx`:**
   - Test SkipToContent component has no violations
   - Verify it renders an anchor with href="#main-content"

   **`input.a11y.test.tsx`:**
   - Test Input with associated label has no violations
   - Render with `<label htmlFor="test">Label</label><Input id="test" />`

   **`select.a11y.test.tsx`:**
   - Test Select with SelectTrigger and SelectContent has no violations
   - Render in open state if possible, or just the trigger

   **`dropdown-menu.a11y.test.tsx`:**
   - Test DropdownMenu with trigger and open content has no violations
   - Use `open` prop if available, or render trigger only

   **`sheet.a11y.test.tsx`:**
   - Test Sheet (open state) with SheetTitle has no violations
   - Similar to Dialog pattern

   Each test file structure:
   ```tsx
   import { describe, it, expect } from "vitest";
   import { render } from "@testing-library/react";
   import { configureAxe } from "vitest-axe";
   import { ComponentUnderTest } from "../component-file";

   const axe = configureAxe({
     impactLevels: ["critical", "serious"],
   });

   describe("ComponentName a11y", () => {
     it("has no critical/serious violations", async () => {
       const { container } = render(<ComponentUnderTest>Content</ComponentUnderTest>);
       expect(await axe(container)).toHaveNoViolations();
     });
   });
   ```
