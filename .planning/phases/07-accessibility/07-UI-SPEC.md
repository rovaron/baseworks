# Phase 7 — UI Design Contract

> Visual and interaction contract for the accessibility retrofit phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | neutral (default style, CSS variables enabled) |
| Component library | Radix UI (via shadcn) |
| Icon library | lucide-react |
| Font | System default (no custom font declared in globals.css) |

**Phase note:** This is an accessibility retrofit phase. No new visual components are introduced. All changes modify existing components and layouts for WCAG 2.1 AA compliance. The design contract below specifies the accessibility-specific visual and interaction requirements.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Skip link padding-y (py-2 = 8px, but inline gap) |
| sm | 8px | Skip link internal spacing |
| md | 16px | Skip link horizontal padding (px-4) |
| lg | 24px | No new usage in this phase |
| xl | 32px | No new usage in this phase |
| 2xl | 48px | No new usage in this phase |
| 3xl | 64px | No new usage in this phase |

Exceptions: Skip link positioning uses `left-4 top-4` (16px from viewport edges) when visible on focus. This follows the existing Tailwind spacing scale.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (normal) | 1.43 (Tailwind default for text-sm) |
| Label | 14px (text-sm) | 500 (medium) | 1.43 |
| Heading | 20px (text-xl) | 600 (semibold) | 1.4 |
| Display | 28px (text-2xl) | 700 (bold) | 1.33 |

**Phase-specific typography rules:**
- Skip link text: 14px (text-sm), weight 400, color `foreground` on `background` surface
- Visually-hidden screen reader text (`sr-only`): Not rendered visually -- uses Tailwind `sr-only` utility (position absolute, width 1px, height 1px, overflow hidden)
- Heading hierarchy: One `h1` per page, `h2` for sections, `h3` for subsections. No skipped levels. Convention only, no lint enforcement (D-03).

---

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | hsl(0 0% 100%) `--background` | Page background, main content area |
| Secondary (30%) | hsl(0 0% 96.1%) `--secondary` | Cards, sidebar (`--sidebar-background`), muted areas |
| Accent (10%) | hsl(0 0% 9%) `--primary` | Primary buttons, active sidebar items, focus rings (`--ring`) |
| Destructive | hsl(0 84.2% 60.2%) `--destructive` | Form validation errors (`role="alert"` text), destructive action buttons |

Accent reserved for: Primary CTA buttons, active navigation items in sidebar, focus ring indicator (`focus-visible:ring-ring`), skip link focus state background.

**Phase-specific color rules:**
- Focus ring color: `--ring` (hsl 0 0% 3.9%) with 2px width and 2px offset. Do not change this existing pattern (D-05).
- Skip link on focus: `bg-background text-foreground` with `shadow-md` and `ring-2 ring-ring` -- high contrast against any page background.
- Form error text: `text-destructive` (already applied to FormMessage). Adding `role="alert"` does not change color.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Skip link label | "Skip to content" |
| Loading state (screen reader) | "Loading..." (sr-only, inside aria-busy container) |
| Loading complete (screen reader) | Content replaces loading text naturally; no explicit "loaded" announcement needed |
| Form validation error | Existing per-field error messages unchanged; `role="alert"` added for immediate screen reader announcement |
| Nav landmark: sidebar | `aria-label="Main navigation"` |
| Nav landmark: header (if present) | `aria-label="User navigation"` (for user menu / breadcrumb nav in header) |
| Main content landmark | `id="main-content"` on `<main>` element |

**No new empty states, CTAs, or destructive actions in this phase.** All copywriting is limited to accessibility labels and screen reader announcements.

## Interaction Contract

This phase introduces interaction patterns that have no visual precedent in the codebase. Each is specified here as the source of truth.

### Skip-to-Content Link

| Property | Value |
|----------|-------|
| Trigger | First Tab keypress on any page |
| Default state | Visually hidden (`sr-only`) |
| Focused state | Fixed position, `left-4 top-4`, `z-50`, rounded-md, bg-background, text-foreground, shadow-md, ring-2 ring-ring |
| Action | Moves focus to `<main id="main-content">` |
| Both apps | Customer app root layout + Admin dashboard root layout |

### Focus Management on Route Change

| Property | Value |
|----------|-------|
| Trigger | Client-side navigation (pathname change) |
| Behavior | Focus moves to `<main id="main-content" tabindex="-1">` |
| First render | Skipped (no focus move on initial page load) |
| Focus outline on main | Suppressed via `focus:outline-none` on the `<main>` element (not an interactive element) |
| Customer app | Custom hook using `usePathname()` from next/navigation |
| Admin dashboard | Custom hook using `useLocation()` from react-router |

### Keyboard Navigation (Existing -- Audit Only)

| Interaction | Expected Behavior | Source |
|-------------|-------------------|--------|
| Tab through page | All interactive elements reachable in DOM order with visible focus ring | D-05 (audit gaps) |
| Escape on dialog/modal | Closes the dialog, returns focus to trigger | Radix Dialog (already implemented) |
| Escape on sheet/drawer | Closes the sheet, returns focus to trigger | Radix Dialog/Sheet (already implemented) |
| Escape on dropdown menu | Closes the dropdown, returns focus to trigger | Radix DropdownMenu (already implemented) |
| Arrow keys in dropdown | Navigate between items | Radix DropdownMenu (already implemented) |

### aria-live Regions

| Region | Politeness | Trigger | Content |
|--------|------------|---------|---------|
| Loading containers | `polite` via `aria-live="polite"` + `aria-busy="true"` | Data fetching starts/completes | `<span class="sr-only">Loading...</span>` while loading; actual content when complete |
| Form validation errors | `assertive` via `role="alert"` (implicit aria-live) | Field validation fails | Existing error message text from FormMessage |
| Toasts | Handled by Sonner | Toast fires | No changes needed (D-09) |

## Semantic Landmark Contract

Specifies the landmark structure for every layout in both apps.

### Customer App Dashboard Layout (`apps/web/app/(dashboard)/layout.tsx`)

```
<SkipToContent />
<SidebarProvider>
  <nav aria-label="Main navigation">
    <Sidebar />
  </nav>
  <SidebarInset>
    <header>
      [SidebarTrigger] [Breadcrumbs]
    </header>
    <main id="main-content" tabindex="-1">
      [Page content with h1]
    </main>
  </SidebarInset>
</SidebarProvider>
```

### Customer App Auth Layout (`apps/web/app/(auth)/layout.tsx`)

```
<SkipToContent />
<main id="main-content" tabindex="-1">
  [Auth form with h1]
</main>
```

### Admin Dashboard Layout (`apps/admin/src/layouts/admin-layout.tsx`)

```
<SkipToContent />
<SidebarProvider>
  <nav aria-label="Main navigation">
    <Sidebar />
  </nav>
  <SidebarInset>
    <header> (already exists)
      [SidebarTrigger] [Breadcrumbs]
    </header>
    <main id="main-content" tabindex="-1"> (already exists, needs id + tabindex)
      [Page content with h1]
    </main>
  </SidebarInset>
</SidebarProvider>
```

### Heading Hierarchy Convention

| Level | Usage | Rule |
|-------|-------|------|
| h1 | Page title | Exactly one per page. Always the first heading in `<main>`. |
| h2 | Major sections | Card titles, form section headings, table section headings |
| h3 | Subsections | Nested content within h2 sections |
| h4-h6 | Not used | Avoid. If needed, indicates content structure is too deep. |
