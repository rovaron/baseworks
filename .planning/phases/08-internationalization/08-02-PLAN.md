For each auth page, add `import { useTranslations } from "next-intl"` and replace every hardcoded string with a translation call. The pattern:

```typescript
// At the top of the component function:
const t = useTranslations("auth");

// For Zod schemas that need translations, define them inside the component
// or use a function that receives t:
```

**IMPORTANT about Zod schemas:** Zod schemas defined at module level (outside components) cannot use hooks. Two approaches:

Option A (recommended): Move the Zod schema inside the component and use `t()` for messages.
Option B: Keep schema at module level with English defaults and override error messages in the form resolver.

Use Option A for simplicity. The schema is small and component-scoped is fine.
