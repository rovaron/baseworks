| Role | Size | Weight | Line Height |
|------|------|--------|-------------|

| Label | 14px (text-sm) | 400 (normal) | 1.43 |

**Phase-specific typography rules:**
- Page title ("Settings"): 24px semibold, matches billing page `h1` pattern
- Section subheadings ("Members", "Pending Invitations"): 20px semibold as `h2`
- Invite accept page org name: 24px semibold, centered
- Invite accept page role badge: 14px normal, uses Badge component
- Member/invitation list items: 14px normal for name/email, 12px (text-xs) muted for metadata

### Invite Dialog

```
<Dialog>
  <DialogHeader>
    <DialogTitle>{t("invite.dialog.title")}</DialogTitle>
    <DialogDescription>{t("invite.dialog.description")}</DialogDescription>
  </DialogHeader>

  <div class="space-y-4">
    <!-- Mode Toggle -->
    <div class="flex items-center justify-between">
      <Label>{t("invite.dialog.modeLabel")}</Label>
      <div class="flex items-center gap-2">
        <span class="text-sm">{t("invite.dialog.emailMode")}</span>
        <Switch checked={isLinkMode} onCheckedChange={setIsLinkMode} />
        <span class="text-sm">{t("invite.dialog.linkMode")}</span>
      </div>
    </div>

    <!-- Email field (hidden in link mode) -->
    {!isLinkMode && (
      <div class="space-y-2">
        <Label>{t("invite.dialog.emailLabel")}</Label>
        <Input type="email" placeholder={t("invite.dialog.emailPlaceholder")} />
      </div>
    )}

    <!-- Role selector (always visible) -->
    <div class="space-y-2">
      <Label>{t("invite.dialog.roleLabel")}</Label>
      <Select>
        <SelectItem value="member">{t("invite.roles.member")}</SelectItem>
        <SelectItem value="admin">{t("invite.roles.admin")}</SelectItem>
      </Select>
    </div>
  </div>

  <DialogFooter>
    <Button variant="outline" onClick={close}>{t("common.cancel")}</Button>
    <Button onClick={submit}>
      {isLinkMode ? t("invite.actions.generateLink") : t("invite.actions.sendInvite")}
    </Button>
  </DialogFooter>
</Dialog>
```

**Post-submit behavior (link mode):** Dialog stays open, shows generated URL in a read-only Input with a CopyLinkButton beside it. User copies the link, then closes dialog manually.

**Post-submit behavior (email mode):** Dialog closes, success toast appears.

### Invite Accept Page (`/invite/[token]`)

```
<div class="flex min-h-screen items-center justify-center bg-background">
  <Card class="w-full max-w-md">
    <CardHeader class="text-center">
      <Avatar class="mx-auto h-16 w-16 mb-4">
        <AvatarFallback>{orgInitials}</AvatarFallback>
      </Avatar>
      <CardTitle class="text-2xl">{orgName}</CardTitle>
      <CardDescription>
        {t("invite.accept.invitedBy", { name: inviterName })}
      </CardDescription>
    </CardHeader>

    <CardContent class="space-y-4 text-center">
      <p>{t("invite.accept.roleAssigned")}</p>
      <Badge variant="secondary">{roleName}</Badge>
    </CardContent>

    <CardFooter class="flex-col gap-3">
      <!-- State 1: Logged in -->
      <Button class="w-full" onClick={accept}>{t("invite.actions.accept")}</Button>
      <Button class="w-full" variant="outline" onClick={decline}>{t("invite.actions.decline")}</Button>

      <!-- State 2: Has account, not logged in -->
      <Button class="w-full" onClick={loginAndAccept}>{t("invite.actions.loginToAccept")}</Button>

      <!-- State 3: No account -->
      <Button class="w-full" onClick={signupAndAccept}>{t("invite.actions.createAccountToJoin")}</Button>
    </CardFooter>
  </Card>
</div>
```

**States:**
1. **Logged in:** Show Accept + Decline Invitation buttons. On accept, call `acceptInvitation()` then `setActive()` then redirect to `/dashboard`.
2. **Not logged in, has account:** Show "Log in to accept" button. Redirects to `/login?invite=[token]`. After login, auto-redirects back to `/invite/[token]`.
3. **No account:** Show "Create account to join" button. Redirects to `/signup?invite=[token]&email=[invitedEmail]`. After signup, auto-redirects back to `/invite/[token]`.
4. **Invalid/expired/already accepted token:** Show error card with message and link to home.
5. **Loading:** Skeleton card matching the accept card dimensions.

## Interaction Contract

### Invite Dialog Flow

| Step | Action | Result |
|------|--------|--------|
| 1 | Click "Invite Team Member" | Dialog opens with email mode selected, role defaults to "Member" |
| 2a | (Email mode) Fill email + select role + click "Send Invite" | API call, dialog closes on success, toast confirms |
| 2b | (Link mode) Select role + click "Generate Link" | API call, dialog shows generated URL with copy button |
| 3b | Click "Copy Link" | URL copied to clipboard, button text changes to "Copied!" for 2 seconds |
| 4b | Close dialog | Dialog closes, pending invitations list refreshes |
| Error | API failure | Toast error, dialog stays open, form preserved |

### Invite Accept Flow

| State | What User Sees | Primary Action |
|-------|---------------|----------------|
| Loading | Skeleton card (avatar + 3 text lines + 2 button skeletons) | Wait |
| Logged in | Org card with Accept Invitation + Decline Invitation | Accept or Decline |
| Not logged in (has account) | Org card with "Log in to Accept" | Redirect to `/login?invite=[token]` |
| No account | Org card with "Create Account to Join" | Redirect to `/signup?invite=[token]&email=[email]` |
| Invalid token | Error card with message + "Go to Home" link | Navigate away |
| Already member | Info card with "You're already a member" + "Go to Dashboard" | Navigate to dashboard |
| Processing accept | Accept button shows Loader2 spinner, Decline Invitation disabled | Wait |
| Accept success | Redirect to `/dashboard` with org auto-selected | Automatic |
| Decline success | Card shows "Invitation declined" + "Go to Home" link | Navigate away |

### Cancel Invitation (Destructive)

| Step | Action | Result |
|------|--------|--------|
| 1 | Click cancel icon on pending invitation row | Confirmation dialog opens |
| 2 | Click "Yes, Cancel Invitation" | API call, dialog closes, row removed from list, toast confirms |
| 3 | Click "Cancel" in dialog | Dialog closes, no action |

### Resend Invite

| Step | Action | Result |
|------|--------|--------|
| 1 | Click resend icon on pending invitation row (email type only) | API call immediately (no confirmation needed) |
| 2 | Success | Toast: "Invitation resent to {email}" |
| 3 | Error | Toast error |

### Remove Member (Destructive)

| Step | Action | Result |
|------|--------|--------|
| 1 | Click "Remove member" in row dropdown | Confirmation dialog opens |
| 2 | Confirm | API call, member removed from list, toast confirms |
| Note | Owner cannot be removed. Current user cannot remove themselves. | These options are hidden, not disabled. |

## Accessibility Notes

- Invite dialog: auto-focuses email input on open (Radix Dialog default)
- Role selector: keyboard navigable via Select component (Radix)
- Members table: semantic `<table>` with proper `<th>` headers and `scope="col"`
- Pending invitations table: same semantic table pattern
- Cancel/Decline Invitation confirmations: Dialog with `role="alertdialog"` and auto-focus on cancel (safe) button
- Toast announcements: handled by Sonner (already configured with aria-live)
- Invite accept page: single `h1` for org name, clear button labels for all three states
- Copy link: `aria-live="polite"` region for "Copied!" feedback
