## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal padding |
| sm | 8px | Compact element spacing, input padding, table cell padding |
| md | 16px | Default element spacing, card padding, form field gaps |
| lg | 24px | Section padding, sidebar item spacing, card group gaps |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page-level vertical spacing, hero sections |
| 3xl | 64px | Page top/bottom margins |

Exceptions: Touch targets for sidebar navigation items and mobile interactions use minimum 44px height (accessibility requirement). This is achieved via `min-h-11` (44px) on interactive sidebar items.

**Source:** Standard 8-point grid. Exception from WCAG 2.5.8 Target Size.

## Typography

| Role | Size | Weight | Line Height | CSS Class |
|------|------|--------|-------------|-----------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` |
| Label | 14px | 500 (medium) | 1.4 | `text-sm font-medium` |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold` |
| Display | 28px | 700 (bold) | 1.2 | `text-2xl font-bold` (page titles only) |

**Rationale:** 14px body is the shadcn default and works well for data-dense admin interfaces and dashboard layouts. 4 sizes provide clear hierarchy without bloat. Both apps share the same typographic scale.

**Font stack:** Tailwind 4 default sans-serif (`ui-sans-serif, system-ui, sans-serif`). No custom web fonts — keeps bundle minimal and load times fast for a starter kit.

**Source:** shadcn/ui defaults. Weight 500 (medium) used for labels per shadcn convention.

## Color

shadcn/ui with `neutral` base color provides the full color token set via CSS custom properties. The 60/30/10 split maps to shadcn's semantic tokens:

| Role | CSS Variable | Approximate Value | Usage |
|------|-------------|-------------------|-------|
| Dominant (60%) | `--background` | hsl(0 0% 100%) | Page background, main content area |
| Secondary (30%) | `--card`, `--muted` | hsl(0 0% 98%) / hsl(0 0% 96%) | Cards, sidebar, table headers, secondary surfaces |
| Accent (10%) | `--primary` | hsl(0 0% 9%) | See reserved-for list below |
| Destructive | `--destructive` | hsl(0 84% 60%) | Delete/deactivate/ban/cancel actions only |

**Accent reserved for (exhaustive list):**
- Primary CTA buttons (e.g., "Subscribe to Plan", "Save Changes", "Sign In")
- Active sidebar navigation item indicator
- Active tab indicator on billing page
- Badge variant for "active" subscription status
- Focus ring on form inputs

**Additional semantic colors:**
- `--success`: Rendered via green Badge variant — healthy queue status, active subscription
- `--warning`: Rendered via yellow Badge variant — degraded system health, approaching usage limit

**Source:** shadcn/ui neutral theme defaults. No custom color overrides — the starter kit uses stock shadcn tokens so downstream forks can reskin easily.

## Interaction States

### Loading States
- **Page-level:** Skeleton components matching the content layout (not spinners)
- **Button submission:** Disabled state + spinner icon inside button during API call
- **Table data:** Skeleton rows (5 rows) while data loads
- **Inline data:** Skeleton matching text width for individual data points

### Error States
- **Form validation:** Inline error messages below each field in `--destructive` color
- **API errors:** Toast notification (Sonner) with error copy from copywriting contract
- **Page-level errors:** Error boundary with centered card showing error message + "Try again" button
- **Network offline:** Toast: "You appear to be offline. Please check your connection."

### Empty States
- Centered within content area
- Icon (from Lucide) + heading + body text + optional CTA button
- Uses `--muted-foreground` for text color

### Destructive Action Pattern
All destructive actions follow the same pattern:
1. User clicks destructive button (red variant: `variant="destructive"`)
2. Dialog opens with warning copy from copywriting contract
3. Two buttons: confirm (destructive variant) and dismiss (outline variant)
4. Confirm triggers API call with button loading state
5. Success: Toast confirmation + data refresh
6. Failure: Toast error message

**Source:** Claude's discretion (loading patterns, error boundaries). Skeleton over spinner per modern convention.

## Design Token Implementation

Tokens are defined in `packages/ui/src/styles/globals.css` using Tailwind 4 CSS-first configuration:

```css
@import "tailwindcss";

@source "../../apps/web/**/*.{ts,tsx}";
@source "../../apps/admin/src/**/*.{ts,tsx}";

@theme {
  --radius-lg: 0.5rem;
  --radius-md: 0.375rem;
  --radius-sm: 0.25rem;
}
```

shadcn's `neutral` base color provides all semantic color tokens (--background, --foreground, --card, --primary, --secondary, --muted, --accent, --destructive, --border, --input, --ring) as CSS custom properties. These are NOT redefined — stock shadcn values are used.

**Source:** RESEARCH.md Pattern 5 (Tailwind 4 Monorepo CSS Architecture), D-19 from CONTEXT.md.
