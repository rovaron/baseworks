**Create `packages/modules/billing/src/adapters/stripe/stripe-webhook-mapper.ts`:**

Maps Stripe webhook event types to NormalizedEvent. The mapping table per RESEARCH.md:

| Stripe Event | NormalizedEventType |
|---|---|
| `checkout.session.completed` | `checkout.completed` |
| `customer.subscription.created` | `subscription.created` |
| `customer.subscription.updated` | `subscription.updated` |
| `customer.subscription.deleted` | `subscription.cancelled` |
| `invoice.payment_succeeded` | `payment.succeeded` |
| `invoice.payment_failed` | `payment.failed` |

The function signature:
```typescript
import type { RawProviderEvent, NormalizedEvent, NormalizedEventType } from "../../ports/types";

const STRIPE_EVENT_MAP: Record<string, NormalizedEventType> = {
  "checkout.session.completed": "checkout.completed",
  "customer.subscription.created": "subscription.created",
  "customer.subscription.updated": "subscription.updated",
  "customer.subscription.deleted": "subscription.cancelled",
  "invoice.payment_succeeded": "payment.succeeded",
  "invoice.payment_failed": "payment.failed",
};

export function mapStripeEvent(rawEvent: RawProviderEvent): NormalizedEvent {
  const normalizedType = STRIPE_EVENT_MAP[rawEvent.type];
  if (!normalizedType) {
    throw new Error(`Unhandled Stripe event type: ${rawEvent.type}`);
  }

  const object = (rawEvent.data as any)?.object ?? rawEvent.data;

  return {
    type: normalizedType,
    providerEventId: rawEvent.id,
    providerCustomerId: (object?.customer as string) ?? "",
    data: {
      subscriptionId: object?.id ?? object?.subscription,
      priceId: object?.items?.data?.[0]?.price?.id,
      status: object?.status,
      currentPeriodEnd: object?.current_period_end
        ? new Date(object.current_period_end * 1000)
        : undefined,
      amount: object?.amount_due ?? object?.amount,
      currency: object?.currency,
    },
    occurredAt: new Date(),
    raw: rawEvent,
  };
}
```

**Create `packages/modules/billing/src/adapters/stripe/stripe-adapter.ts`:**

Implements PaymentProvider by wrapping Stripe SDK calls. Move all logic from existing command files into this adapter. Key implementation details:

- Constructor takes `{ secretKey: string; webhookSecret: string }` and creates a `new Stripe(secretKey, { typescript: true })` instance
- `readonly name = "stripe"`
- `createCustomer`: calls `stripe.customers.create()` with metadata.tenantId and idempotencyKey (from `on-tenant-created.ts` logic)
- `createCheckoutSession`: calls `stripe.checkout.sessions.create()` with mode "subscription" (from `create-checkout-session.ts`)
- `createOneTimePayment`: calls `stripe.checkout.sessions.create()` with mode "payment" (from `create-one-time-payment.ts`)
- `cancelSubscription`: calls `stripe.subscriptions.update()` with `cancel_at_period_end: true` (from `cancel-subscription.ts`)
- `changeSubscription`: retrieves subscription, then updates with new price (from `change-subscription.ts`)
- `getSubscription`: calls `stripe.subscriptions.retrieve()` and maps to ProviderSubscription
- `createPortalSession`: calls `stripe.billingPortal.sessions.create()` (from `create-portal-session.ts`), returns `{ url: session.url }`
- `verifyWebhookSignature`: calls `stripe.webhooks.constructEvent()`, returns `{ id: event.id, type: event.type, data: event.data }`
- `normalizeEvent`: delegates to `mapStripeEvent()` from the webhook mapper
- `getInvoices`: calls `stripe.invoices.list()` and maps each invoice (from `get-billing-history.ts`)
- `reportUsage`: calls `stripe.subscriptionItems.createUsageRecord()` (from `sync-usage.ts`)

All methods use `{ idempotencyKey: crypto.randomUUID() }` where applicable (preserving existing D-09 pattern).
