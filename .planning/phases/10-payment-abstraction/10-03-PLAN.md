**Step 2: Create `packages/modules/billing/src/adapters/pagarme/pagarme-webhook-mapper.ts`:**

Maps Pagar.me webhook events to NormalizedEvent per RESEARCH.md mapping table:

| Pagar.me Event | NormalizedEventType |
|---|---|
| `subscription.created` | `subscription.created` |
| `subscription.canceled` | `subscription.cancelled` |
| `charge.paid` | `payment.succeeded` |
| `charge.payment_failed` | `payment.failed` |
| `order.paid` | `checkout.completed` |

```typescript
import type { RawProviderEvent, NormalizedEvent, NormalizedEventType } from "../../ports/types";

const PAGARME_EVENT_MAP: Record<string, NormalizedEventType> = {
  "subscription.created": "subscription.created",
  "subscription.canceled": "subscription.cancelled",
  "charge.paid": "payment.succeeded",
  "charge.payment_failed": "payment.failed",
  "order.paid": "checkout.completed",
};

export function mapPagarmeEvent(rawEvent: RawProviderEvent): NormalizedEvent {
  const normalizedType = PAGARME_EVENT_MAP[rawEvent.type];
  if (!normalizedType) {
    throw new Error(`Unhandled Pagar.me event type: ${rawEvent.type}`);
  }

  const data = rawEvent.data as any;

  return {
    type: normalizedType,
    providerEventId: rawEvent.id,
    providerCustomerId: data?.customer?.id ?? "",
    data: {
      subscriptionId: data?.subscription?.id ?? data?.id,
      priceId: data?.plan?.id,
      status: data?.status,
      currentPeriodEnd: data?.current_period_end
        ? new Date(data.current_period_end)
        : undefined,
      amount: data?.amount ?? data?.last_transaction?.amount,
      currency: data?.currency ?? "BRL",
    },
    occurredAt: new Date(),
    raw: rawEvent,
  };
}
```

**Step 3: Create `packages/modules/billing/src/adapters/pagarme/pagarme-adapter.ts`:**

Implements PaymentProvider using Pagar.me SDK (or REST API). Key differences from Stripe:

- `readonly name = "pagarme"`
- Constructor takes `{ secretKey: string; webhookSecret: string }`
- All Pagar.me API calls use `sk_xxxx` as Basic Auth header or SDK init
- `createCustomer`: POST /customers with name, email, metadata
- `createSubscription`: POST /subscriptions with plan_id, customer_id, payment_method
- `cancelSubscription`: DELETE /subscriptions/{id}
- `changeSubscription`: PATCH /subscriptions/{id} with new plan_id
- `getSubscription`: GET /subscriptions/{id}
- `createOneTimePayment`: POST /orders with items, payments (supports Pix + credit card)
- `createCheckoutSession`: POST /orders (Pagar.me doesn't have hosted checkout like Stripe -- create an order and return a payment URL or checkout page URL). Return `{ sessionId: order.id, url: order.checkouts?.[0]?.payment_url ?? "" }`
- `createPortalSession`: return `null` (Pagar.me has no customer portal equivalent -- per RESEARCH.md pitfall 1)
- `verifyWebhookSignature`: Validate webhook using HMAC-SHA256 with webhook secret. If the SDK provides a verification method, use it. Otherwise, compare `crypto.timingSafeEqual` of the HMAC of the raw body against the signature header. NEVER use simple string comparison (timing attack risk).
- `normalizeEvent`: delegates to `mapPagarmeEvent()` from the webhook mapper
- `getInvoices`: GET /charges?customer_id={id} and map to ProviderInvoice shape
- `reportUsage`: not implemented (return undefined -- method is optional on interface)

If the SDK is unavailable or problematic, implement using raw fetch calls to Pagar.me REST API v5 (https://api.pagar.me/core/v5/):
```typescript
private async request(method: string, path: string, body?: unknown) {
  const response = await fetch(`https://api.pagar.me/core/v5${path}`, {
    method,
    headers: {
      "Authorization": `Basic ${btoa(this.config.secretKey + ":")}`,
      "Content-Type": "application/json",
    },
    body: body ? JSON.stringify(body) : undefined,
  });
  if (!response.ok) throw new Error(`Pagar.me API error: ${response.status}`);
  return response.json();
}
```

**Step 2: Update `packages/modules/billing/src/provider-factory.ts`:**

Add Pagar.me support to the factory:

```typescript
import { env } from "@baseworks/config";
import type { PaymentProvider } from "./ports/payment-provider";
import { StripeAdapter } from "./adapters/stripe/stripe-adapter";
import { PagarmeAdapter } from "./adapters/pagarme/pagarme-adapter";

let providerInstance: PaymentProvider | null = null;

export function getPaymentProvider(): PaymentProvider {
  if (!providerInstance) {
    const providerName = env.PAYMENT_PROVIDER ?? "stripe";

    switch (providerName) {
      case "stripe":
        providerInstance = new StripeAdapter({
          secretKey: env.STRIPE_SECRET_KEY!,
          webhookSecret: env.STRIPE_WEBHOOK_SECRET!,
        });
        break;
      case "pagarme":
        providerInstance = new PagarmeAdapter({
          secretKey: env.PAGARME_SECRET_KEY!,
          webhookSecret: env.PAGARME_WEBHOOK_SECRET!,
        });
        break;
      default:
        throw new Error(`Unknown payment provider: ${providerName}`);
    }
  }
  return providerInstance;
}

export function resetPaymentProvider(): void {
  providerInstance = null;
}

export function setPaymentProvider(provider: PaymentProvider): void {
  providerInstance = provider;
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External webhook -> Pagarme adapter | Pagar.me webhook payloads validated via HMAC-SHA256 |
| API -> Pagar.me REST API | Outbound calls with secret key via Basic Auth |
| Env config -> Factory | PAYMENT_PROVIDER drives adapter selection at startup |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-07 | Spoofing | Pagar.me webhook verification | mitigate | HMAC-SHA256 signature verification using `crypto.timingSafeEqual` -- never simple string comparison (timing attack). |
| T-10-08 | Information Disclosure | Pagar.me secret key | mitigate | Key passed only to adapter constructor, authenticated via Basic Auth header, never logged. |
| T-10-09 | Denial of Service | Missing env validation | mitigate | Zod superRefine enforces PAGARME_SECRET_KEY when PAYMENT_PROVIDER=pagarme, preventing runtime crash on first billing call (RESEARCH.md pitfall 4). |
| T-10-10 | Tampering | Provider switch at runtime | accept | Provider singleton created once at startup. No mechanism for runtime switch. Requires restart. Low risk. |
</threat_model>
