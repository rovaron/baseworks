## 1. Local adapter HMAC signing scheme (SC#4 / D-25-01)

### 1.1 Secret source
- Env var: **`STORAGE_LOCAL_SIGNING_SECRET`**.
- **Dev default** (when unset): the literal `"baseworks-dev-insecure-local-signing-secret"`. Acceptable because the Local adapter is prod-banned (`validateStorageEnv` throws on `STORAGE_PROVIDER=local && NODE_ENV=production`).
- **Future env.ts requirement (SPEC ONLY — DO NOT edit `env.ts` in Phase 25):** when `STORAGE_PROVIDER=local` and `NODE_ENV !== "test"`, `validateStorageEnv()` SHOULD require `STORAGE_LOCAL_SIGNING_SECRET` to be set (named-var crash on missing), mirroring the existing per-provider checks. Phase 25 documents this; the change is implemented in a later plan/phase. The dev default keeps the local test suite green today.
- Secret is read inside the signing helper via `process.env.STORAGE_LOCAL_SIGNING_SECRET ?? DEV_DEFAULT`. Never logged.

### 1.2 Canonical string to sign (EXACT)
Newline-joined, **no trailing newline**, fields in this exact order. Two shapes keyed by operation:

- **Upload (PUT):**
  ```
  PUT\n{bucket}\n{key}\n{exp}\n{max}
  ```
- **Read (GET):**
  ```
  GET\n{bucket}\n{key}\n{exp}
  ```

Where:
- `{bucket}` = raw `StorageBucket` string.
- `{key}` = raw `StorageKey` string (the un-encoded key; URL-encoding is applied only when building the path, never inside the canonical string).
- `{exp}` = integer Unix epoch **seconds** at which the URL expires = `Math.floor(Date.now()/1000) + expiresInSec`.
- `{max}` = integer `maxByteSize` (upload only).

The PUT and GET canonical forms differ in both the method prefix AND the field count, so an upload signature can never be replayed as a read signature or vice-versa.

### 1.3 Signature
```
sig = createHmac("sha256", secret).update(canonical, "utf8").digest("hex")
```
`node:crypto`. Lowercase hex.

### 1.4 URL shape (EXACT)
Path: `/api/files/local/{encodeURIComponent(bucket)}/{key-encoded}` where the key is encoded per-segment (split on `/`, `encodeURIComponent` each segment, re-join with `/`) so the key's `/` separators survive but `?`, `#`, spaces, etc. are escaped.

Query params (in this order):
- **Upload:** `?exp={exp}&max={maxByteSize}&sig={sig}`  → returned as `SignedUpload { method: "PUT", url, expiresAt }` (no `fields`, no `headers`).
- **Read:** `?exp={exp}&sig={sig}`  → returned as `SignedRead { url, expiresAt }`.

`expiresAt = new Date(exp * 1000).toISOString()`.

**Pitfall 1 resolution (locked):** the result object carries NO `key`/`storageKey` field — only `url`/`method`/`expiresAt`. The key DOES appear inside the URL path because the local endpoint must address the object; this is acceptable because (a) in Phase 26 the key carries a mandatory unguessable `nanoid(24)` segment, and (b) the HMAC `sig` makes the URL unforgeable and tamper-evident. The forbidden thing — a standalone raw-key field in the typed result — stays absent (asserted by conformance behavior #7/#8).

### 1.5 Expiry + verification (`verifyLocalSignature`)
Phase 25 ships the verifier (unit-tested now; consumed by the Phase 26 endpoint). Signature:
```ts
verifyLocalSignature(args: {
  method: "PUT" | "GET";
  bucket: string;
  key: string;
  exp: number;        // parsed from query
  max?: number;       // required for PUT, absent for GET
  sig: string;        // hex from query
  now?: number;       // epoch seconds; defaults to Date.now()/1000 (test seam)
}): { valid: true } | { valid: false; reason: "expired" | "bad_signature" | "malformed" }
```
Steps (in order):
1. Validate shape: `exp` is a finite integer; PUT requires integer `max`; `sig` is non-empty hex. Else `malformed`.
2. Expiry: if `now > exp` → `expired`. (Checked BEFORE signature so an expired-but-valid sig still fails.)
3. Recompute `expected = hmac(canonical)`.
4. Constant-time compare: decode both `sig` and `expected` from hex to `Buffer`; if lengths differ → `bad_signature` (do NOT call `timingSafeEqual` on unequal lengths — it throws); else `crypto.timingSafeEqual(a, b)` → `valid` / `bad_signature`.

### 1.6 Local object I/O (filesystem)
Root = `process.env.STORAGE_LOCAL_PATH ?? "./storage"`. Object path = `{root}/{bucket}/{key}`.
- **Path-traversal guard:** reject any `bucket`/`key` whose normalized resolved path escapes `{root}` (contains `..` segments or resolves outside root) → throw a clear error. Applies to every I/O method.
- `putObject` writes bytes to `{root}/{bucket}/{key}` (mkdir -p parent) AND writes a sidecar `{root}/{bucket}/{key}.meta.json` = `{ "mimeType": "<mime>" }` so `stat` can return an authoritative `mimeType` (the filesystem stores none). Use `Bun.write` / `node:fs/promises`.
- `getObject` reads the data file → `Uint8Array`; missing file → throw.
- `stat` → `{ byteSize: <fs size of data file>, mimeType: <sidecar mimeType> }`; missing data file → `null`. `etag`/`lastModified` optional (lastModified MAY be the fs mtime ISO string; etag omitted).
- `delete` removes the data file AND the sidecar; missing files → resolve without throwing (idempotent).

## 2. Shared conformance suite (SC#1, SC#2 / D-25-04)

**File:** `packages/storage/src/adapters/__tests__/conformance.ts` (named `.ts`, NOT `.test.ts`, so `bun test` does not execute it standalone — it is imported by the adapter test files).

**Exported signature (LOCKED):**
```ts
import type { FileStorage } from "../../ports/file-storage";

export function runFileStorageConformance(
  label: string,
  makeStorage: () => FileStorage | Promise<FileStorage>,
): void;
```
- Opens `describe(\`FileStorage conformance — ${label}\`, () => { ... })`.
- Backend-agnostic: uses ONLY the `FileStorage` port surface.
- Each `test()` uses a fresh unique key derived from `crypto.randomUUID()` (NOT `Math.random`); `afterEach`/`afterAll` deletes any key it wrote so reruns stay clean. Round-trip body = the committed `baseline-100x100.png` fixture bytes via the fixtures loader (binary, exercises non-text bytes).
- Bucket: a fixed conformance bucket name, e.g. `"conformance"` (created on demand for FS; pre-created for MinIO by CI bootstrap).

**Behaviors asserted (LOCKED — one `test()` each):**
1. **name discriminator** — `storage.name` is a non-empty string (adapter identity present for log lines).
2. **round-trip byte-equality** — `putObject` then `getObject` returns bytes identical in length and content (`Buffer.compare === 0`), including the binary PNG fixture.
3. **stat returns authoritative byteSize + mimeType** — after a put of `N` bytes with `mimeType: "image/png"`, `stat` returns `byteSize === N` and `mimeType === "image/png"`.
4. **stat of missing key returns `null`** — `stat` on a never-written key resolves to `null` (never throws).
5. **delete removes** — after `put`→`delete`, `stat` returns `null` AND `getObject` rejects.
6. **delete of missing key is idempotent** — `delete` on a never-written key resolves without throwing.
7. **signUpload returns a PUT url with expiry** — `signUpload({ ..., expiresInSec: 600 })` → `method === "PUT"`, `url` is a non-empty string, `expiresAt` parses to a valid Date in the future, and the result has NO `storageKey`/`key` property (`expect("storageKey" in r).toBe(false)`; same for `"key"`).
8. **signRead returns a url with expiry** — `signRead({ ..., expiresInSec: 600 })` → `url` non-empty, `expiresAt` valid & future, no raw-key property.
9. **expiry honors expiresInSec** — `expiresAt` ≈ `now + expiresInSec` within ±5s tolerance (catches off-by-1000 / ms-vs-s bugs).

Tests 1, 7, 8, 9 require no live backend (pure URL minting). Tests 2–6 require a live backend; for Local that is the filesystem (always available), for S3/S3-compat the skip wrapper in the caller gates them.

## 4. CORS template requirements (SC#3)

`validateCorsConfig(json)` parses the AWS S3 `{ "CORSRules": [ { ... } ] }` shape. R2/MinIO/Garage all accept this shape. For EACH rule it asserts (each violation → an entry in `errors[]`, `ok = errors.length === 0`):
- **No wildcard origins:** every `AllowedOrigins` entry must be a concrete origin — **error if any entry contains the character `"*"`** (rejects both bare `"*"` and `"*.subdomain"` forms in committed templates). Committed templates use concrete origins only: `"https://app.example.com"` + `"http://localhost:3000"`.
- **ETag exposed:** `ExposeHeaders` must contain `"ETag"` (case-insensitive compare).
- **PUT present:** `AllowedMethods` must contain `"PUT"`. (Warn — non-fatal — if `GET`/`HEAD` absent.)
- **Structural:** `CORSRules` is a non-empty array; each rule has non-empty `AllowedOrigins`, `AllowedMethods`, and `ExposeHeaders` arrays.

All four templates (`aws-s3`, `r2`, `minio`, `garage`) MUST satisfy the above and additionally include `AllowedHeaders` covering `content-type`, `content-length`, `x-amz-checksum-sha256`, `x-amz-meta-tenant-id` (per Pitfall 11). `AllowedMethods` = `["PUT","POST","GET","HEAD"]`. `MaxAgeSeconds: 3600`. Per-backend addressing nuances (MinIO/Garage path-style) are documented in the Phase 31 integration doc, not encoded in JSON.

## 5. Fixtures (SC#5) — deterministic generation

All produced by `scripts/generate-fixtures.ts`. **No `Math.random`.** PNGs are built by hand: IHDR + IDAT(`deflateSync` of raw scanlines, each row prefixed with filter byte `0x00`) + IEND, with a deterministic CRC-32 over each chunk. Pixels come from pure functions of `(x, y)` so output is byte-identical on every run. `deflateSync` from `node:zlib` (level fixed, e.g. `level: 9`).

| Fixture | How generated (deterministic) |
|---|---|
| `baseline-100x100.png` | 100×100, every pixel a fixed color (e.g. RGB `(220,38,38)`). Smallest valid PNG for round-trip. |
| `photo-5000x5000.png` | 5000×5000 RGB gradient `r=(x*255/4999)`, `g=(y*255/4999)`, `b=((x^y)&255)` — procedural "photo"; compresses to a modest size. |
| `bomb-50000x50000.png` | 50000×50000 single-color (RGB grayscale). Scanlines are all-identical so `deflateSync` collapses 2.5e9 pixels to a few KB on disk while declaring a 50000×50000 IHDR (true decompression bomb). Build IDAT by deflating the repeated row pattern. |
| `truncated.png` | First `floor(len*0.4)` bytes of `baseline-100x100.png` (valid signature+IHDR, IDAT cut mid-stream). |
| `svg-with-script.svg` | Static string: `<svg xmlns="http://www.w3.org/2000/svg" width="100" height="100"><script>alert('xss')</script><rect .../></svg>`. |

After writing artifacts, the script computes sha256 of each and writes `manifest.json`. `fixtures.test.ts` re-hashes the committed files and asserts equality with the manifest — this both proves reproducibility and detects accidental hand-edits. The loader `loadFixture(name)` reads the file as `Uint8Array`; conformance uses `baseline-100x100.png`; Phase 28 transform tests consume all five.

### Risks

- **R5 — presign `responseContentDisposition`:** the port accepts `responseContentDisposition` on `signRead`; Bun `presign` exposes `type` (response-content-type) — confirm a content-disposition option before relying on it. - **R6 — Local sidecar `.meta.json`:** the sidecar must not be mistaken for a real object by Phase 26 listing logic; flag for Phase 26. Phase 25 `stat`/`delete` handle the pair atomically.
- **R7 — Bomb fixture size:** the 50000×50000 IHDR must be honored by sharp/imagescript in Phase 28; if the single-color deflate is rejected as malformed by a decoder, regenerate with a minimal real row pattern. Phase 25 only stores/loads the bytes.
- **R8 — Migration/worktree noise:** no new migrations in Phase 25 (schema is Phase 24); ensure no stray `.test.ts` under `__tests__` runs the conformance helper standalone (named `conformance.ts`, not `*.test.ts`).
