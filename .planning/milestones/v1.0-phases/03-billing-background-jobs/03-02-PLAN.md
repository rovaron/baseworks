1. Create `packages/modules/billing/src/routes.ts` per D-10, D-13, Pitfall 1, Pitfall 5:
   - Export an Elysia plugin function `billingRoutes`
   - The webhook route MUST be excluded from tenant middleware and auth (D-13). It should be mounted in the module's route group that runs before tenant middleware, OR use a separate Elysia group. Since billing routes will include both public (webhook) and tenant-scoped routes, split them:

   Webhook endpoint implementation:
   ```typescript
   import { Elysia } from "elysia";
   import { getStripe } from "./stripe";
   import { env } from "@baseworks/config";
   import { createDb } from "@baseworks/db";
   import { webhookEvents } from "@baseworks/db";
   import { eq } from "drizzle-orm";
   import { createQueue } from "@baseworks/queue";

   // Lazy queue initialization
   let webhookQueue: Queue | null = null;
   function getWebhookQueue() {
     if (!webhookQueue && env.REDIS_URL) {
       webhookQueue = createQueue("billing:process-webhook", env.REDIS_URL);
     }
     return webhookQueue;
   }

   export const billingRoutes = new Elysia({ prefix: "/api/billing" })
     .post("/webhooks", async (ctx) => {
       const stripe = getStripe();
       const sig = ctx.request.headers.get("stripe-signature");
       if (!sig) return new Response("Missing stripe-signature header", { status: 400 });

       // CRITICAL: Use raw body text for signature verification (Pitfall 1)
       const rawBody = await ctx.request.clone().text();

       let event;
       try {
         event = stripe.webhooks.constructEvent(
           rawBody,
           sig,
           env.STRIPE_WEBHOOK_SECRET!,
         );
       } catch (err) {
         return new Response("Webhook signature verification failed", { status: 400 });
       }

       // Idempotency check (D-11)
       const db = createDb(env.DATABASE_URL);
       const existing = await db.select()
         .from(webhookEvents)
         .where(eq(webhookEvents.stripeEventId, event.id))
         .limit(1);

       if (existing.length > 0) {
         return { received: true };
       }

       // Insert event record with status "pending"
       await db.insert(webhookEvents).values({
         stripeEventId: event.id,
         eventType: event.type,
         status: "pending",
         payload: JSON.stringify(event.data),
       });

       // Enqueue for async processing (D-10, Pitfall 5 -- return 200 fast)
       const queue = getWebhookQueue();
       if (queue) {
         await queue.add("process-webhook", {
           eventId: event.id,
           type: event.type,
         }, { jobId: event.id }); // BullMQ-level dedup via jobId
       }

       return { received: true };
     });
   ```
   
   IMPORTANT: The webhook route must receive the raw body. Elysia may auto-parse JSON bodies. Use `ctx.request.clone().text()` to get the raw body string before any parsing. If Elysia's body parsing interferes, set the route to parse: { body: "text" } or equivalent Elysia config.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Stripe -> webhook endpoint | External HTTP requests from Stripe servers; must verify signature |
| webhook endpoint -> BullMQ | Verified event data crosses from HTTP to job queue |
| billing module -> Stripe API | Outbound API calls with secret key |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Spoofing | Webhook endpoint | mitigate | `stripe.webhooks.constructEvent()` verifies HMAC signature with timing-safe comparison; reject if signature missing or invalid (400 response) |
| T-03-05 | Tampering | Webhook replay | mitigate | webhook_events table with unique constraint on stripe_event_id prevents duplicate processing; BullMQ jobId set to event.id for queue-level dedup |
| T-03-06 | Repudiation | Webhook processing | mitigate | All webhook events logged to webhook_events table with payload, status, and processedAt timestamp for audit trail |
| T-03-07 | Information Disclosure | Stripe secret key | mitigate | STRIPE_SECRET_KEY and STRIPE_WEBHOOK_SECRET stored in env vars only; lazy initialization in stripe.ts; never logged or returned in responses |
| T-03-08 | Elevation of Privilege | Webhook endpoint auth bypass | mitigate | Webhook endpoint excluded from tenant/auth middleware (D-13) but protected by Stripe signature verification; no user-facing billing operations possible through webhook endpoint |
| T-03-09 | Tampering | Webhook event ordering | mitigate | lastEventAt column in billing_customers; webhook processor only updates if event.created > lastEventAt (Pitfall 3 from RESEARCH.md) |
</threat_model>
