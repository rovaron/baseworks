<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| worker process <-> Redis | Worker connects to Redis; connection URL from env |
| job data <-> processor | Job payloads are internal-only (enqueued by our code) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | Redis connection string | mitigate | REDIS_URL stored in env var only; never logged or returned in API responses; pino logger configured to redact URLs |
| T-03-02 | Denial of Service | BullMQ worker | accept | Worker concurrency capped at 5 by default; BullMQ has built-in stalled job detection; low risk for internal job processing |
| T-03-03 | Tampering | Job payload injection | accept | Jobs are only enqueued by trusted internal code (CQRS handlers, webhook processor); Redis is not publicly accessible |
</threat_model>
